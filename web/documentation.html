Database / Inheritance strategy

Our database is built on MySQL using JPA with EclipseLink to achieve Object Relational Mapping.

First we created the MySQL database with SQL. We thought, as recommended by the teachers, that it would be the proper approach to generate the entities from the database. However, since MySQL does not support inheritance, we had to change our approach towards building the database and entities.

In our second attempt we created the Entities first with the appropriate annotations for achieving the correct entity relations in the database. After creating the Entity-classes, we generated the database from these. This solved our problem, since we could add inheritance when writing in Java with JPA Annotations. It would have been nice to have known this from the start, since both strategies have been backed by different teachers.

Returns a person with the given phone number as a JSON-object. If a person exists with the given phone number you will receive a response with the status code 200 (OK). Otherwise our web service will catch a custom exception and return the appropriate error message in JSON-format.

Delete Person

DELETE http://PROJECTNAME/api/person/4

Deletes a person with the given phone number. If the person exists in the database, the web service will respond with an HTTP status code 200 (OK). If the person does not exist, the web service will throw a custom exception which will return the appropriate error message in JSON-format.
